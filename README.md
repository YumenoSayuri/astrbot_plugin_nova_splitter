# Nova 全能引擎 (Nova-Omni)

> 专为辉宝主人定制的终极护卫：绝对空回防御 + 消息分段 + 引导思考 + 睡眠联动 (v1.4.0)

## 核心特性

### 🚀 0. 绝对空回防御与火力重装 (v1.4.0 新增核心)
当底层大模型由于各种原因返回空内容时，系统将在底层拦截网进行绝对防御：
- **无限火力重试**：基于原始请求参数，原地向大模型发起连续重试（次数面板可配置）。
- **备用支援接管**：若原模型彻底断联，可配置触发自动切换到“备用模型提供商”接力回答，确保会话永不卡死。

### 1. 多种分段策略

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| `char_count` | 按字数均分 | 需要均匀分段时 |
| `punctuation` | 按标点分段 | 保持语义完整性 |
| `llm_assist` | LLM智能分段 | 复杂文本，需要语义理解 |
| `hybrid` | 混合模式 | 先按标点，不够再按字数 |

### 2. 智能标点清理

| 清理模式 | 效果 |
|----------|------|
| `edge_only` | 仅清理分段边界标点，保留句内标点 |
| `replace_space` | 指定标点替换为空格 |
| `preserve` | 完全保留所有标点 |
| `custom` | 自定义正则规则 |

### 3. 引导思考 (v1.2.0+)

让AI从"有问必答的客服"变成"有自己想法、会思考要不要搭理你的真人"。

**工作原理：**
- 双重注入：system prompt 追加完整引导 + 用户消息后追加简短强调提醒
- AI 会先在 `<thought>` 标签内进行内心独白（思考要不要回、回什么）
- 插件自动拦截思维链内容，用户只看到真正的回复（多重兼容：缺少开头`<`、只有闭合标签等）
- 管理员可通过 `想啥呢` 指令查看AI最近的思考内容
- 无思维链时自动清空缓存，不会显示过期旧内容

**沉默机制：**
- 如果AI思考后决定不回复，在回复部分输出 `[沉默]` 等关键词
- 插件检测到沉默标记后完全拦截回复，实现真正的"已读不回"

**思维链转发：**
- 开启后每次提取到思维链都会实时转发到指定群聊/私聊
- 转发格式：`在群123456 2026年3月23日20:30:17思考了：思考内容`
- 方便管理员实时监控AI在不同群的思考过程

### 4. 睡眠模式与 sexlife 联动定时睡眠

自动读取 sexlife 插件的日程数据，根据日程中的睡觉时段自动切换睡眠/唤醒状态。

**工作原理：**
- 每天定时一次读取 sexlife 的 `schedule_data.json`（不是轮询，零开销）
- 从 timeline 中识别标题包含"梦乡"/"睡觉"/"睡前"等关键词的时段
- 当前时间在睡觉时段内 → 自动休眠；不在 → 自动唤醒
- 支持跨午夜时段（如 22:00-08:00）

### 5. 睡眠模式 (v1.2.0 新增)

管理员可以通过指令让AI进入/退出睡眠状态。

| 指令 | 效果 |
|------|------|
| `快去睡觉` | AI进入休眠，拦截所有LLM请求（可在行为管理加别名） |
| `快点起床` | 唤醒AI，恢复正常响应（可在行为管理加别名） |

## 配置说明

### 基础分段配置

```yaml
split_mode: char_count        # 分段模式
target_segments: 3            # 目标分段数
max_length_no_split: 50       # 不分段阈值
punctuation_clean_mode: edge_only  # 标点清理模式
```

### 引导思考配置

```yaml
enable_thought_guide: false   # 是否启用引导思考
thought_guide_prompt: "..."   # 引导思考提示词（含示例，可自定义）
thought_tag: thought          # 思维标签名
thought_forward_enabled: false  # 是否转发思维链到指定目标
thought_forward_target: ""    # 转发目标 UMO（如 aiocqhttp:FriendMessage:QQ号）
silence_keywords: "沉默,静默,不想回复,不回复,不想搭理,没必要回复"  # 沉默关键词
```

### 睡眠模式配置

```yaml
enable_sleep_mode: false      # 是否启用睡眠模式功能
sleep_whitelist_groups: ""    # 睡眠白名单群号（逗号分隔，群聊需@bot才放行）
sleep_whitelist_users: ""     # 睡眠白名单QQ号（逗号分隔，私聊直接放行）
sleep_full_silence: false     # 完全静默（拦截所有消息发送，包括其他插件回复）
```

### sexlife 联动配置

```yaml
sleep_auto_schedule: false    # 是否启用 sexlife 联动
sleep_schedule_read_time: "00:05"   # 每日定时读取日程的时间
sleep_schedule_keywords: "梦乡,睡觉,入睡,睡眠,深度睡眠,睡前,晚安"  # 睡觉时段关键词
```

### 延迟配置

```yaml
delay_strategy: linear
linear_base: 0.3
linear_factor: 0.05
```

## 管理员指令

| 指令 | 说明 | 权限 |
|------|------|------|
| `想啥呢` / `在想啥` | 查看AI最近一次的思维链内容 | 管理员 |
| `快去睡觉` | 让AI进入休眠（可在行为管理加别名） | 管理员 |
| `快点起床` | 唤醒AI（可在行为管理加别名） | 管理员 |
| `读取日程` | 手动读取sexlife日程并更新睡眠状态 | 管理员 |

## 使用方法

1. 将插件放入 `AstrBot/data/plugins/` 目录
2. 重启 AstrBot
3. 在控制台配置插件参数
4. 启用 `enable_thought_guide` 开始使用引导思考
